# Junction — 에이전시 탄생 스토리

> 출처: Claude 대화 기록 (2026-04-09, "1인 AI 네이티브 브랜드 에이전시 첫 고객 준비")
> 정리일: 2026-07-03

## 이름의 유래

처음엔 "JUN"이었다가 → **"Junction"**으로 확정. "연결/교차로"라는 단어 의미에서 착안.

**슬로건**: "Connect Every Brand Experience"

## 초기 비주얼 방향
화이트 배경, 미니멀, 클린. 포인트 컬러로 라임그린(#C8FF00) 사용.

## 포지셔닝 — 에이전시 vs 스튜디오

디자인 스튜디오는 소규모·크래프트 중심·특정 분야의 결과물 완성도와 저자성에 집중하는 반면, 디자인 에이전시는 전략·마케팅·브랜드컨설팅·미디어까지 통합해서 더 넓은 범위로 운영되고 PM·전략가·마케터 등 다양한 역할이 섞인 팀 구조에, 클라이언트의 측정 가능한 비즈니스 성과 지향성이 강함.

Junction은 **AI 네이티브 특성 + 소규모로 풀서비스를 제공할 수 있는 능력** 덕분에 에이전시 모델에 가까움. 다만 브랜드 아이덴티티가 크래프트/저자성 쪽으로 기울면 스튜디오 포지셔닝도 어울림 — 아직 열린 질문.

## 초기 프로토타입: HALU 케이스스터디

첫 랜딩페이지 제작 시 가상의 포트폴리오 케이스로 프리미엄 스킨케어 브랜드 **"HALU"**(한국어 "하루"에서 착안)를 만들어 브랜드 컨셉, 로고 배리에이션, 컬러 팔레트, 타이포그래피, CSS 목업, 브랜드 적용 사례를 전부 구축. Netlify Drop으로 배포.

## AI 네이티브 툴스택 방향 (2026-03-31 대화 기준)

1인 에이전시의 툴 구성을 3계층으로 정리:
- **메인 생성 엔진**: Midjourney 또는 Nano Banana
- **특수목적 툴**: ChatGPT Image
- **후처리/통합 레이어**: Adobe Firefly

힉스필드 같은 애그리게이터 플랫폼보다 **각 모델의 특성을 직접 파악해서 쓰는 것**이 에이전시 경쟁력에 유리하다는 관점 — Claude, Figma, 프롬프트 역량을 이미 보유한 걸 전제로 한 실용적 조합.

## 구상했던 자동화 파이프라인 (미완성 아이디어)

"브랜드 에셋 자동화 팩토리" — Google Sheets + Make.com + Claude + Gemini + Google Drive 연동.
- Router로 "신규"(10개 프롬프트) / "변주"(5개 프롬프트) 모드 분기
- Iterator → Gemini 이미지 생성 → Drive 저장 → 에셋 라이브러리 로깅(Row ID 기반 변주 lookup)

→ 세컨드브레인(방주) 자체가 사실상 이 구상의 진화된 버전. 당시엔 Make.com 기반 자동화였는데, 지금은 Claude Code + GitHub 기반 방주로 방향이 바뀜.
