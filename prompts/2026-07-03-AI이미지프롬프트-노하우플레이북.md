# AI 이미지 생성 프롬프트 — 노하우 플레이북

> 출처: Claude 대화 기록 다수 (2025-11 ~ 2026-06, 롯데카드/롯데호텔/하나투어/뉴케어/KANU 작업 전반)
> 반복 시행착오에서 얻은 패턴만 압축 정리
> 정리일: 2026-07-03

## 기본 스타일 원칙 (전 클라이언트 공통)

- 화이트 배경, 정면/직교 구도, 텍스트 없음
- 최대 3~4개 컬러 팔레트, 포토리얼리스틱 8K
- 여행/라이프스타일 계열은 필름 사진 감성 (Kodak Portra 400, Fuji Pro 400H)
- 크롭 방지를 위한 충분한 여백 확보
- 작은 사이즈에서도 인식 가능하도록 단순한 구성 유지

## 반복 검증된 노하우

**1. 부정문 프롬프트의 역설**
"no text, no lines" 같은 부정 프롬프트는 오히려 AI가 그 요소를 생성하게 만드는 경우가 많음. → **긍정 서술어만 사용**하는 게 안전 ("plain white background" O, "no text background" X)

**2. 트리거 단어 회피**
"coupon", "movie ticket" 같은 특정 단어는 원치 않는 장식 텍스트를 유발함. → "solid plain rectangular card" 같은 중립적 표현으로 대체

**3. 복잡한 합성 구도는 축소 스케일에서 혼란 유발**
멀티 오브젝트 합성보다 단순 구성이 썸네일 사이즈에서 더 잘 읽힘

**4. 브랜드 패키지 텍스트/라벨은 AI가 정확히 재현 못 함**
→ **2단계 워크플로우**: (1) 대략적인 형태/컬러로 베이스 이미지 생성 → (2) 실제 패키지를 인페인팅/합성으로 후처리. 특히 한글 브랜드 패키지는 신뢰도가 매우 낮음.

**5. "미니멀하게"를 프롬프트 길이로 착각하지 말 것**
"더 미니멀하게" 요청이 오면 프롬프트 문장을 줄이는 게 아니라 **장면 자체의 복잡도**를 줄여야 함. 프롬프트는 오히려 구체적일수록 의도한 결과가 나옴 (원하는 결과가 단순해도 디테일한 지시는 유지).

**6. 손(hand) 디테일은 별도로 강조 필요**
가족/인물 라이프스타일 사진에서 손가락·그립 디테일이 뭉개지는 경우多. 카메라/렌즈 스펙(Hasselblad X2D 100C, Sony A7R V 등)을 명시하면 개선됨.

**7. 콘텐츠 모더레이션 회피**
특정 단어 조합(예: 어린 가족구성원 + "커팅보드" 근접 묘사)이 필터에 걸릴 수 있음 → 해당 조합 자체를 피하거나 대상 묘사를 덜 구체적으로.

**8. 참조 이미지 의존도는 %로 명시**
"이 참조 이미지 90% 반영해줘" 처럼 구체적 비율을 언급하면 AI가 스타일 반영 정도를 더 잘 조절함.

**9. 업스케일링은 별도 모델/프롬프트로 분리**
"제미나이 나노바나나프로" 같은 업스케일 전용 모델은 원본 생성 프롬프트와 다른, 훨씬 단순한 해상도 강화 전용 프롬프트를 써야 함.
