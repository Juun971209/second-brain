# UI/UX 컴포넌트 용어집

> 출처: Claude 대화 기록 여러 건 (2026-06-30 ~ 2026-07-01)
> 클라이언트 작업 시 참고용 컴포넌트 네이밍 레퍼런스
> 정리일: 2026-07-03

## 버튼/그룹 계열

| 상황 | 명칭 |
|---|---|
| 버튼들이 간격 없이 붙어있음 | **Segmented Control** |
| 버튼들 사이에 마진(간격) 있음 | **Button Group** |
| 입력창 안에 내장된 액션 버튼 (예: "중복확인") | Inline Button / Trailing Button / Suffix Button / Input Adornment / Action Input Field (Material Design 기준) |

**형태 기준 "OO버튼" 네이밍 패턴**: 텍스트버튼, 플로팅버튼(FAB), 아웃라인버튼, 고스트버튼(Ghost Button) 등

## 카드/박스 계열

| 상황 | 명칭 |
|---|---|
| 이벤트 페이지의 혜택/perk 표시 박스 | **카드(Card)** / **베네핏 카드(Benefit Card)** |
| 카드를 감싸는 상위 구조 | Section |
| 카드 내부 요소 | Badge, Title, Description/Caption |
| 여러 혜택 카드를 리스트로 묶을 때 | 혜택 카드 리스트 (Benefit Card List) |

## 입력/텍스트 계열

| 영문 | 한글 표기 |
|---|---|
| Notification | 알림 |
| Text Field | 입력창 / 입력 필드 |
| Text Input (한 줄) | 텍스트 입력란 |
| Text Area (여러 줄) | 여러 줄 입력란 |
| Breakpoint | 분기점 / 기준점 |

**실무 팁**: 실제 UI/UX 작업에서는 영문 용어를 그대로 쓰는 경우가 많고, 한글 대응어는 문서화·설명용으로 예비해두는 경우가 대부분.

## 타이포그래피 계열

**그로테스크(Grotesque)**
- 타이포그래피 의미: 초기 산세리프 서체 분류 (Neo-Grotesque, Geometric, Humanist와 함께 사용)
- 디자인/예술 전반의 미학적 의미로도 사용됨

---
## 활용 방법
새 클라이언트 작업 시작할 때 "이 용어집 기준으로 컴포넌트 이름 잡아줘" 하면 일관된 네이밍 유지 가능. Junction 프로젝트 전반의 디자인 시스템 문서화 시 재사용.
