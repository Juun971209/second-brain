# 운영디자인_JUN — 에셋 관리 시스템 구조

> 출처: Google Drive "운영디자인_JUN" 스프레드시트
> Junction 내부 AI 이미지 에셋 생성/관리 워크플로우
> 정리일: 2026-07-03

## 시트 구조 2개

### 1. 관제탑 (컨트롤 타워)
신규 생성이 필요한 이벤트/캠페인을 등록하는 큐. 컬럼: 진행상태(신규/변주) · 이벤트 내용 · 대상 번호

예시 항목:
- 거친 돌 질감의 프리미엄 비누 패키지 디자인 (신규)
- 호텔 최대 7% 적립 또는 12개월 무이자 (신규)
- 데이터 무제한 가입 이벤트 (신규)
- 자동차보험료 조회 및 신규가입 이벤트 (신규)

### 2. 에셋_라이브러리
실제 생성된 이미지 프롬프트를 기록하는 아카이브. 컬럼: 에셋번호 · 생성일시 · 관제탑 ID(참조) · 사용 프롬프트(전문) · 이미지 링크 · 프로젝트 폴더 · 선택여부 · 최종파일 ID · 최종완료여부

## 워크플로우 패턴

```
관제탑에 이벤트 등록 (신규/변주 구분)
        ↓
해당 ID로 여러 개 프롬프트 변주 생성 (보통 5개 내외)
        ↓
에셋_라이브러리에 전체 프롬프트 + 이미지 링크 기록
        ↓
선택된 것만 "선택" 컬럼 체크 + 최종파일 ID 기록
```

**"신규" vs "변주" 구분**이 핵심 — 완전히 새로운 컨셉인지, 기존 성공 프롬프트의 변형인지 분리 관리. 이건 4월 대화에서 구상했던 "브랜드 에셋 자동화 팩토리"(Make.com 기반)의 실제 운영 버전.

## 실제 프롬프트 패턴 예시 (자동차보험 이벤트)

가중치 문법 활용: `(Solid pure white background:1.5)`, `(No text:1.5)`, `(Top-down view, flat lay, perfectly centered, symmetrical:1.3)`, `(Ultra-realistic, 8k resolution:1.2)`, `(Muted colors, max 2 colors:1.2)`

같은 오브젝트(유리병+모래+코르크마개, 또는 자동차 모형)를 각도·조명·질감만 바꿔가며 5개 변주로 생성하는 패턴이 반복됨.
