# KFC 앱 현황 분석

> 출처: Notion — KFC UX 스터디 "자유 기록" 데이터베이스
> 작성자: 준형 | 정리일: 2026-07-03

## 앱 히스토리

- **2017년**: 앱 출시 (쿠폰·멤버십·KFC Pay 통합), 앱스토어 ID id1255799839
- **2015년**: KFC 배달 서비스 도입
- **2020년 8월**: 멤버십 1차 개편
- **2021년 12월**: 멤버십 2차 개편
- **2023년**: 오케스트라PE 인수 후 디지털 전략 강화
- **2024년**: 업데이트 12회 이상 (매번 "사용자 편의성 개선"만 반복, 구체적 내용 공개 없음)
- **2025년 5월**: 차세대 앱 전면 개편 계획 발표 (노후 UI 공식 인정)

## 핵심 기능 구조

**주문 채널**
- 징거벨오더(스마트오더): 앱 사전 주문·결제 후 매장 픽업 — KFC 자사 핵심 채널
- 딜리버리: 매장별 운영 여부 상이
- 포장은 징거벨오더/매장에서만, 콜센터는 딜리버리 전용

**혜택 채널**
- 쿠폰: 앱 전용/프로모션/멤버십 등급별 다양한 종류, 중복 사용 불가, 채널별(POS/딜리버리/스마트오더) 적용 상이
- 치킨나이트: 매일 21~22시 1+1, 앱·자사 주문 전용 (배달 플랫폼 미적용)
- 멤버십(콜킬포인트): 5,000원당 1포인트, 등급별(실버/골드/VIP) 혜택

**결제 수단**
- KFC Pay(자체 간편결제), KFC 선불카드

**핵심 페인포인트 연결지점**
앱 핵심 가치가 "쿠폰 + 징거벨오더"인데, 이 두 기능의 연결 동선이 끊겨있음. 쿠폰 확인 후 스마트오더로 자연스럽게 이어지지 않아 중간 이탈 발생.

## 실적/트래픽 데이터 (연도순)

| 시점 | 지표 |
|---|---|
| 2024.12 | MAU 68만명 (패스트푸드 앱 9위, 스타벅스 773만·버거킹 267만·맥도날드 208만 대비 크게 낮음) |
| 2024 | 누적회원 240만 돌파, 앱 매출 28% 성장 |
| 2024 | 매출 2,923억(+18%), 영업이익 164억(5.7배↑), 매장당 일평균매출 +16%, 방문객 +11% |
| 2025.05 | MAU 118만명(8위), 6개월만 74%↑ (버거킹 308만의 절반 미만) |
| 2025 Q1 | 멤버십 263만명(+30%), 신규가입 3월 +117% |
| 2025 상반기 | 매출 1,678억(+19.2%), 앱 다운로드 월평균 10.6만건(+79%), 누적회원 286만 |
| 2025년 | 연매출 3,780억(+29.3%), 영업이익 247억(1.5배↑), 동일매장 매출 +25% — 가격인상이 아닌 제품·마케팅·디지털 전략이 주요인 |

**해석**: 매출/실적은 2년 연속 역대 최대 기록 중인데, 앱 UX 품질(평점 1.7점)은 그 성장 속도를 따라가지 못하는 괴리가 뚜렷함. 디지털 전환 잠재력이 UX 문제로 발목 잡히는 구조.

## 앱스토어 평점

- KFC Korea: **1.7점** (5점 만점, 960개 평가 기준) — 주요 패스트푸드 앱 중 최하위
- 맥도날드 3.8점, 버거킹 3.5점 대비 극단적으로 낮음
- 1점대 리뷰 비율 전체의 60% 이상

## 정책 관련 특이사항 (나무위키)

- 쿠폰 중복 사용 불가 (키오스크에서 2번째 바코드 스캔 시 거부 메시지)
- 치킨나이트(1+1)는 앱/자체 주문 전용, 배달 플랫폼 앱 주문 미적용
- 쿠폰 정책 복잡도가 사용자 혼란의 근본 원인 중 하나
