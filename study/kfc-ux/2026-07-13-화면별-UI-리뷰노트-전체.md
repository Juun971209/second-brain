# KFC 앱 — 화면별 UI 리뷰노트 (홈~멤버십/카드)

> 출처: [Notion — 4. 팀 내부 UI 리뷰노트](https://app.notion.com/p/f1d7a6492f32822fa3fa81a95751806e?pvs=1)

> 참고: 로그인 화면 리뷰는 [[2026-07-12-팀-UI-리뷰노트]]에 이미 정리됨. 이 노트는 그 이후 추가된 홈화면~멤버십/카드 섹션을 정리.

## 요약

팀원(준형·은지·지현)이 KFC 앱 화면을 직접 살펴보며 남긴 코멘트가 로그인 화면 외에도 홈화면, 스마트오더(딜리버리/징거벨오더), 쿠폰, 멤버십/카드까지 전 화면으로 확장됨. 반복적으로 지적된 것은 여백/정렬 불일치, VOC와 동일한 배달 추적 부재·매장 자동배정 문제, 그리고 주문취소 버튼이 실제로는 취소가 안 되는 구조적 문제.

## 홈화면

### 메인
**화면 흐름**: 앱 진입 → 메인(홈) 화면 — 상단 배너, 딜리버리/징거벨오더 진입 버튼

| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 메인 | 상단 로고와 중복 배치 — 이벤트배너에서 KFC 로고 삽입 위치 고려 필요 | 준형 |
| 메인 | 하단 플로팅 바코드 버튼 클릭 시 멤버십/카드 화면으로 넘어감 — 왜 있는지 불분명 | 준형 |

### 장바구니
**화면 흐름**: 메인 → 장바구니 진입 → 딜리버리/징거벨오더 탭 전환 → (비어있을 시) 안내 팝업

| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 장바구니 | 메뉴 바로 삭제 기능 없음 — 선택 후 삭제해야 해서 번거로움 (👍2) | 남지현 |
| 장바구니 | 매장정보 없을 시 매장 검색 화면으로 강제 이동 — 딜리버리 탭에서 매장 정보 의존성 높음 | 준형 |

## 스마트오더/딜리버리

전체 화면 흐름: 딜리버리(신규주소 등록) → 딜리버리(기존 주소) → 딜리버리/메뉴선택 → 딜리버리/선택메뉴 주문 → 딜리버리/장바구니 → 주문완료

### 딜리버리(신규주소 등록)
| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 주소확인·주소찾기 | "배달받을 주소 입력" 필드가 있는데 별도로 "새로운 주소" 창이 또 있어 중복됨 — 둘 다 같은 주소찾기 화면으로 이동 | 은지 |
| 주소확인 | 주소를 입력하고 주문을 진행하지 않으면 저장이 안 됨 (별도 저장 기능도 없음) — VOC 리뷰와 동일 문제 | 남지현 |
| 주소확인 | 배달매장 라벨 얼라인 체크 필요 | 준형 |

### 딜리버리(기존 주소)
| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 주소확인 | 무조건 KFC가 지정해준 매장에서만 배달 시킬 수 있음 — VOC 리뷰(딜리버리 카테고리)와 동일 문제 | 남지현 |
| 이전 주소 목록 팝업 | 취소버튼이 있는데 굳이 X 버튼도 별도로 필요한지 의문 | 준형 |

### 딜리버리/메뉴선택
카테고리 탭(추천메뉴/치킨/버거/사이드/음료) → 메뉴 리스트 → 치킨나이트 프로모션 페이지

| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 추천메뉴 탭 | 사이드 탭만 아이콘이 일러스트풍 — 탭 간 아이콘 스타일 불일치 | 남지현 |
| 카테고리 탭 | 버튼 뒤로 이미지 비치는 것이 가시성을 떨어뜨림 | 은지 |
| 버거 탭 | 메뉴 선택→단품/세트→재료추가→옵션추가→음료추가가 한 페이지에 안 보이고 페이지별 분리돼 뎁스가 길어짐 — VOC와 동일 지적 | 남지현 |
| 치킨나이트 프로모션 페이지 | 21시~22시 사이에 뜨는 페이지에 이벤트 시간·주문방법 공지·설명 부족 | 준형 |

### 딜리버리/선택메뉴 주문
| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 주문하기·결제수단 | 간편결제에 네이버페이·카카오페이 등 주요 수단 부재 — KFC PAY·PAYCO·L.pay만 제공 | 준형 |

### 주문완료
결제하기 → 주문완료(접수 확인, 주문·조리·배달·완료 진행상태 표시) → 주문내역 확인

| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 주문완료 | 배달 시작 후 배달원 실시간 위치 파악 불가 (👍1) | 남지현 |
| 주문완료 | 매장 주소·배달 주소가 지도에 함께 표시되면 좋겠음 — 주문 실수 방지(쿠팡이츠 참고) (👍1) | 은지 |
| 주문완료 | 주문완료 화면 애니메이션 완성도 아쉬움 | 은지 |

## 스마트오더/징거벨오더

### 징거벨오더/주문매장결정
매장 선택(가까운 매장/이전 주문 매장/검색하기) → 매장확인(자동매칭, 수령매장) → 매장 위치(지도) 확인

| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 매장확인·이전 주문 매장·검색하기(확인 버튼) | 비활성화 버튼 색이 화면마다 다름 | 은지 |
| 매장 위치(지도) | 선택된 매장만 보여짐 — 전체 매장 위치를 한번에 볼 수 있으면 좋겠다는 의견 | 은지 |
| 매장 선택·검색하기 | 마이>매장찾기 지도와 UI/데이터가 다름 | 준형 |

### 징거벨오더/메뉴선택
| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 상단 배너 문구("이제 줄 서지 마세요! 딩동~") | 위트있는 라이팅이 전체적으로 정직한 라이팅 톤과 안 맞아 뜬금없게 느껴짐 (👍1) | 준형 |
| 메뉴선택 진입 과정 | 메뉴 확인 과정이 너무 복잡함(수신동의 팝업, 매장정보 등 거쳐야 함). 마이>메뉴확인도 모웹으로 넘어감 — VOC와 동일 지적 | 준형 |

### 징거벨오더/주문하기
| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 주문하기 전체 | 단계별(매장정보·캠페인·주문정보·결제수단) 섹션 간 여백값 차이가 크지 않아 구분이 잘 안됨 — 텍스트 위계도 체크 필요 | 준형 |

### 징거벨오더/주문완료
| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 주문취소 버튼 | 누르면 실제 취소는 안 되고 "매장에서 해주세요" 안내만 뜸 — 버튼을 왜 만들었는지 의문 (👍2) | 은지, 준형 |

## 쿠폰

### 매장방문전용 쿠폰
| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 쿠폰 목록·모두 받기 | "쿠폰 모두 받기"와 개별 선택 발급의 차이가 불분명 | 준형 |
| 쿠폰 상세(바코드 팝업)·제목 | 할인가 텍스트에 강조 효과가 없고, 텍스트가 영역을 초과해 읽기 힘듦 | 준형 |
| 쿠폰 상세·유효기간 | "유효기간: OOOO.OO.OO까지"보다 "유효기간 ~일 남음" 표기가 더 직관적일 듯 | 준형 |
| 쿠폰 상세·닫기 버튼 | 딤 처리와 그레이 배경이 겹쳐서 "닫기" 텍스트만 둥둥 떠보임 | 준형 |
| 쿠폰 목록 | "사용하기" 버튼 추가해 바로 주문으로 이어지면 좋겠음 | 준형 |

### 징거벨오더전용 쿠폰
| 화면 | 이슈 | 발견자 |
| --- | --- | --- |
| 쿠폰 상세(바코드 팝업)·제목 | 할인율보다 할인된 실제 가격을 더 강조해주면 좋겠음 (👍2) | 남지현 |
| 쿠폰 상세·바코드 | 스마트오더용 쿠폰인데 바코드가 왜 필요한지 지적 많음. 딜리버리/매장취식/스마트오더용 쿠폰 구분이 탭으로만 되어있어 처음 쓰는 사람은 헷갈릴 수 있음 (👍1) | 남지현 |
