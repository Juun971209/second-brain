## 요약

앱스토어 리뷰를 딜리버리·쿠폰·앱 사용성 전반 3개 카테고리로 나눠 패턴 분석. 딜리버리는 리뷰 8건 중 6건이 배달 추적·안내 부재를 지적(최다 클러스터). 쿠폰은 6건 전원이 "쿠폰 받기→실제 주문 적용" 동선 단절을 지적 — 기존 `페인포인트-리뷰데이터.md`에서 확인한 인사이트와 동일한 결론이지만 원본 리뷰·패턴 분류가 훨씬 상세해짐. 앱 사용성 전반은 18건으로 가장 다양한데, 강제 업데이트 문제(6건)와 다국어(영어) 미지원(4건, 2019~2026년 5년 넘게 반복)이 가장 큰 두 축.

## 1. 딜리버리 (리뷰 8건 중 6건이 배달 추적·안내 부재 지적)

**패턴 1 · 배달 추적·안내 부재 (8건 중 6건)**
- 예상 도착시간 미표기 — 언제 올지 알 수 없어 "마냥 기다려야" 함
- 라이더 실시간 위치 확인 불가 — 최소 1~5분 전 알림이라도 필요하다는 요청
- 조리중·배달중·완료 상태 알림이 없어 계속 새로고침 필요
- 실제 사례: 주문 후 45분 만에 배달중 확인, 54분 만에 수령. 완료 알림도 없었음
- 극단적 사례: 결제 버튼 눌렀는데 화면이 멈춰서 안 된 줄 알고 배민으로 재주문 → 양쪽 다 배달 완료돼서 이중 결제. 배달 상태 안내가 전혀 없었던 게 원인

**패턴 2 · 매장 선택 불가**
- 배달매장을 직접 고르지 못하고 자동배정됨 — 원하는 매장에서 배달 주문 불가

**패턴 3 · 쿠폰-딜리버리 연동 오류**
- 딜리버리 주문 시 쿠폰이 적용 안 된 채로 바로 주문이 진행됨
- 주문 취소 요청해도 고객센터에서 취소 불가 안내 — 대응 미흡

**패턴 4 · 기타 UX 디테일**
- 라이더 요청사항이 저장되지 않아 매번 재입력 필요
- 배달내역 상세보기가 부실함 (위치·요청사항·쿠폰적용 여부 확인 불가)
- 메뉴 비교 시 X 버튼 누르면 장바구니 전체가 초기화됨

## 2. 쿠폰 관련 (6건 전원이 쿠폰-주문 연결 동선 문제 지적)

기존 페인포인트-리뷰데이터.md에서 확인한 "쿠폰-주문 연결 단절" 인사이트와 결론이 일치하며, 경쟁사(버거킹·맥도날드) 대비 명시적 비교가 리뷰 2건에서 확인됨.

**패턴 1 · 쿠폰-주문 연결 동선 단절 (6건 중 6건, 전원 지적)**
- 쿠폰을 받아도 바로 적용되지 않고, 메뉴를 직접 찾아 담은 뒤 결제 직전에야 쿠폰을 적용해야 함
- 버거킹·맥도날드는 쿠폰 다운로드 후 바로 사용 가능한데 KFC만 이 흐름이 없음 (경쟁사 대비 명시적 비교 2건)
- 메뉴를 잘못 고르면 결제창까지 갔다가 처음부터 다시 시작해야 하고, 입력 정보가 전부 삭제됨

**패턴 2 · 징거벨오더 쿠폰 사용성 문제**
- 징거벨오더 쿠폰은 징거벨오더 시에만 사용 가능한데, 누르면 바코드만 뜨는 구조라 실제 적용 방법을 알기 어려움
- 이상적인 흐름(쿠폰 확인 → 팝업으로 바로 상품 담기)이 아니라, 사용자가 메뉴명을 "기억"해서 홈으로 이동 후 직접 담아야 함

**패턴 3 · 쿠폰 정렬·탐색 불편**
- 받은 쿠폰이 리스트 가장 아래로 내려가서 스크롤해서 찾아야 함
- 매장 키오스크에서도 쿠폰 메뉴가 결제 단계가 아닌 별도 위치에 배치돼 있어 헷갈림

**패턴 4 · 주문 확인 불가**
- 징거벨오더 주문 후 앱을 재실행하면 주문번호를 확인할 수 없음 (주문 내역은 있지만 번호가 안 보임)
- 앱을 끄면 주문번호 자체가 사라짐 — 타 앱은 종료해도 유지됨

## 3. 앱 사용성 전반 (18건, 가장 다양한 이슈)

**패턴 1 · 강제 업데이트 문제 (6건, 최다 클러스터)**
- 앱을 킬 때마다 업데이트를 요구, 이틀에 한 번꼴로 업데이트 발생
- "최신버전이 아닙니다" 뜨면서 접속 자체가 차단됨, 업데이트 이후 오히려 접속 불가 상태 발생
- "닭 튀기는 앱이 왜 은행·증권앱보다 더 철저한 보안을 요구하냐"는 비판

**패턴 2 · 다국어(영어) 미지원 (4건, 2019~2026년까지 5년 넘게 반복)**
- 언어 변경 옵션 자체가 없음 — 한국어 못하는 외국인 다수라는 지적 (2019, 2021, 2025, 2026 리뷰에 반복)

**패턴 3 · 로그인·인증 장벽**
- 웹사이트 회원가입 후 로그인 시 IP차단 페이지 발생, 앱과 고객문의 채널에서도 동일 오류 재현

**패턴 4 · 주문 단계 비효율 (클릭 수·주소 재입력)**
- 메뉴 검색만 하려 해도 주소를 매번 재입력해야 함, 뒤로가기 시 처음부터 다시 입력
- 장바구니 담기까지 클릭 수가 지나치게 많음

**패턴 5 · 적립·알림권한 UX**
- 적립 바코드를 메인에서 바로 확인할 수 없어 매번 사 먹어도 적립을 놓침
- 알림 권한을 켜지 않으면 앱 사용 자체가 불가 — "데이터 수집용 아니냐"는 의심

**패턴 6 · 이벤트 시점 서버 불안정** — 이벤트 알림 보낼 때마다 서버 접속 장애 발생

**패턴 7 · 업데이트發 기능 오류** — 업데이트 이후 딜리버리 기능 자체가 작동 안 함(배달가능매장 없음 오류) → 타 앱으로 이탈
