# KFC 앱 페인포인트 — 리뷰 데이터 & 벤치마킹

> 출처: Notion — KFC UX 스터디 "자유 기록" 데이터베이스
> 정리일: 2026-07-03

## 리뷰 & VOC (사용자 불만)

**핵심 반복 불만 5가지** (앱스토어 리뷰 / brunch.co.kr 분석글 2024.03)
1. 쿠폰 어디서 쓰는지 모르겠다
2. 결제 직전 쿠폰 적용 실패
3. 카카오페이·네이버페이 등 메이저 간편결제 미지원
4. 회원가입 시 생년월일 오류
5. 기프티콘 사용 불가

**리뷰 패턴**: 별점 1~2점 리뷰 중 "쿠폰 쓰려다 그냥 포기했다"는 후기 다수. 단순 버그보다 **UX 동선 문제**로 보임 (앱스토어 리뷰 2024~2025).

→ 문제 유형이 개별 버그가 아니라 구조적으로 UX 설계 단계에 집중되어 있음.

## 직접 사용성 테스트

**쿠폰 → 스마트오더 흐름 테스트**
쿠폰 화면에서 "사용" 버튼을 눌렀는데 주문 화면으로 연결되지 않고 메인으로 튕겨나감. 메뉴 선택 → 장바구니 → 쿠폰 적용 순서를 처음부터 반복해야 했음.

→ 이 단절이 리뷰에서 반복되는 "쿠폰 포기" 불만의 실제 원인으로 확인됨 (VOC와 직접 테스트 결과가 일치).

## 경쟁사 벤치마킹

**맥도날드 앱**
쿠폰 선택 시 해당 메뉴가 자동으로 장바구니에 담기고 주문 흐름으로 바로 연결됨. KFC와 달리 쿠폰→주문 동선이 끊기지 않음.

→ KFC 개선 방향의 직접적 레퍼런스 후보.

## 종합 인사이트

- VOC(간접) + 사용성테스트(직접) + 벤치마킹(비교) 세 가지 데이터가 **동일한 지점**(쿠폰-주문 연결 단절)을 가리키고 있어 페인포인트의 확실성이 높음
- 개선 방향은 "쿠폰 선택 즉시 주문 흐름 자동 연결" — 맥도날드 패턴 참고 가능
- 간편결제 미지원, 회원가입 오류 등은 별도 트랙의 부가 개선 과제로 분리 가능
