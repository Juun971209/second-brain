# KFC 앱 플로우차트 (딜리버리·징거벨오더·쿠폰·통합)

## 핵심 요약
팀원이 Miro에서 만든 상세 플로우를 참고해 분기·재시도 루프를 반영, 딜리버리/징거벨오더/쿠폰 3개 플로우와 이를 하나로 연결한 통합 플로우까지 mermaid로 정리한 자료. "4. 팀 내부 UI 리뷰노트" 문서 하단에 추가된 섹션으로, 쿠폰-주문 연결 단절(2순위 페인포인트)을 구조적으로 검증하는 근거 자료이기도 함.

## 출처
- Notion: [4. 팀 내부 UI 리뷰노트](https://app.notion.com/p/f1d7a6492f32822fa3fa81a95751806e) > 플로우차트 섹션

## 상세 내용

### 딜리버리
```mermaid
flowchart TD
    Home[홈] --> Delivery[딜리버리 진입]
    Delivery --> AddrType{신규 주소 or 기존 주소}
    AddrType -->|신규| AddrSearch[주소찾기: 검색 → 결과선택 → 확정]
    AddrType -->|기존| AddrList[이전 주소 목록에서 선택]
    AddrSearch --> StoreCheck{배달가능 매장 있음?}
    AddrList --> StoreCheck
    StoreCheck -->|No| StoreFail[배달불가 안내 팝업] --> AddrType
    StoreCheck -->|Yes| MenuCat[메뉴 카테고리 선택<br/>추천/치킨/버거/사이드/음료]
    MenuCat --> MenuPick[메뉴 선택]
    MenuPick --> SetType{세트 or 단품}
    SetType -->|세트| SetOptions[재료·통다리·프렌치프라이·음료 옵션 선택]
    SetType -->|단품| Upgrade{세트 업그레이드?}
    Upgrade -->|Yes| SetOptions
    Upgrade -->|No| Cart
    SetOptions --> Cart[장바구니 확인<br/>메뉴추가·옵션수정 가능]
    Cart --> Order[주문하기<br/>배달정보·주문정보 확인 · 쿠폰/선불카드/배달비 상세 확인 가능]
    Order --> PayMethod[결제수단 선택<br/>KFC PAY/PAYCO/L.pay/카드]
    PayMethod --> Pay[결제 진행]
    Pay --> PayResult{결제 성공?}
    PayResult -->|실패| PayMethod
    PayResult -->|성공| Done[주문완료<br/>접수 확인]
    Done --> Status[주문현황 진행단계<br/>주문→조리→배달→완료]
```

### 징거벨오더
딜리버리와 별개로 홈에서 바로 진입하는 독립된 흐름.
```mermaid
flowchart TD
    Home[홈] --> ZEntry[징거벨오더 진입]
    ZEntry --> StoreType{가까운 매장 유무}
    StoreType -->|Yes| StoreAuto[가까운 매장 자동 선택]
    StoreType -->|No| StoreSearch[매장명 검색 → 검색결과에서 매장 선택]
    StoreAuto --> StoreConfirm[매장확인 · 매장위치 지도]
    StoreSearch --> StoreConfirm
    StoreConfirm --> MenuCat[메뉴 카테고리 선택<br/>추천/치킨/버거/사이드/음료]
    MenuCat --> MenuPick[메뉴 선택]
    MenuPick --> SetType{세트 or 단품}
    SetType -->|세트| SetOptions[재료·통다리·프렌치프라이·음료 옵션 선택]
    SetType -->|단품| Upgrade{세트 업그레이드?}
    Upgrade -->|Yes| SetOptions
    Upgrade -->|No| Cart
    SetOptions --> Cart[장바구니 확인<br/>메뉴추가·옵션수정 가능]
    Cart --> Order[주문하기<br/>매장정보·수령방법 선택 · 쿠폰/선불카드/배달비 상세 확인 가능]
    Order --> PickupType{수령방법}
    PickupType -->|바로방문| Arrive[매장 도착 확인]
    PickupType -->|예약방문| Reserve[예약방문 시간 선택]
    Arrive --> Coupon[쿠폰 선택 팝업]
    Reserve --> Coupon
    Coupon --> PayMethod[결제수단 선택]
    PayMethod --> Pay[결제 진행]
    Pay --> PayResult{결제 성공?}
    PayResult -->|실패| PayMethod
    PayResult -->|성공| Done[주문완료<br/>대기번호 발급]
    Done --> Cook[조리 시작 알림] --> Receipt[전자영수증 확인] --> CancelTry{주문취소 시도}
    CancelTry -->|Yes| CancelInfo[안내 팝업: 매장에서 취소 요청]
    CancelTry -->|No| Complete[완료: 픽업 준비/수령 알림]
```

### 쿠폰
```mermaid
flowchart TD
    Coupon[쿠폰 메인] --> Tab1[매장방문전용 탭]
    Coupon --> Tab2[징거벨오더전용 탭]
    Coupon --> Tab3[딜리버리전용 탭]
    Coupon --> Tab4[쿠폰입력 탭]

    Tab1 --> T1a[쿠폰 모두 받기 / 개별 받기]
    T1a --> T1b[발급 완료 팝업]
    T1b --> T1c[쿠폰 상세 · 바코드 확인]
    T1c --> T1d[닫기]

    Tab2 --> T2a[쿠폰 모두 받기]
    T2a --> T2b[쿠폰 상세 · 바코드 확인]
    T2b --> T2c[닫기]

    Tab3 --> T3a[쿠폰 모두 받기]
    T3a --> T3b[쿠폰 상세 확인]
    T3b --> T3c[닫기]

    Tab4 --> T4a[쿠폰번호 입력]
    T4a --> T4b[등록]
```

### 통합 플로우 (딜리버리·징거벨오더·쿠폰)
표준 순서도 도형(ANSI/ISO) 규칙 참고: 터미널(양끝 둥근 도형) = 시작/끝, 마름모 = 판단, 직사각형 = 처리, 평행사변형 = 입력/출력(검색·번호입력). `C_Stock -.주문 시 사용 가능.-> Pay` 구간이 점선인 것에 주목 — 쿠폰-주문 연결이 설계상 존재하지만 발견성이 낮다는 2순위 페인포인트의 구조적 근거.
```mermaid
flowchart TD
    Home([홈]) --> Choice{서비스 선택}
    Choice -->|딜리버리| D_AddrType{신규 주소 or 기존 주소}
    D_AddrType -->|신규| D_Search[/주소 검색/]
    D_Search --> D_Result[검색결과 선택]
    D_Result --> D_Confirm[주소 확정]
    D_AddrType -->|기존| D_List[이전 주소 목록에서 선택]
    D_List --> D_Confirm
    D_Confirm --> D_StoreCheck{배달가능 매장 있음?}
    D_StoreCheck -->|No| D_Fail[배달불가 안내]
    D_Fail --> D_AddrType
    D_StoreCheck -->|Yes| MenuCat[메뉴 카테고리 선택]
    Choice -->|징거벨오더| Z_StoreType{가까운 매장 유무}
    Z_StoreType -->|Yes| Z_Auto[가까운 매장 자동 선택]
    Z_StoreType -->|No| Z_Search[/매장명 검색/]
    Z_Search --> Z_Result[검색결과에서 매장 선택]
    Z_Auto --> Z_Confirm[매장확인 · 위치 지도]
    Z_Result --> Z_Confirm
    Z_Confirm --> MenuCat
    Choice -->|쿠폰| C_Main[쿠폰 메인<br/>매장방문/징거벨오더/딜리버리/쿠폰입력 탭]
    C_Main --> C_TabType{탭 유형}
    C_TabType -->|매장방문·징거벨오더·딜리버리| C_Get[쿠폰 받기: 전체/개별]
    C_Get --> C_Detail[쿠폰 상세 확인]
    C_Detail --> C_Close[닫기]
    C_TabType -->|쿠폰입력| C_Input[/쿠폰번호 입력/]
    C_Input --> C_Register[등록]
    C_Close --> C_Stock([보관함에 저장])
    C_Register --> C_Stock
    C_Stock -.주문 시 사용 가능.-> Pay
    MenuCat --> MenuPick[메뉴 선택]
    MenuPick --> SetType{세트 or 단품}
    SetType -->|세트| Options[옵션 선택<br/>재료·통다리·프렌치프라이·음료]
    SetType -->|단품| Upgrade{세트 업그레이드?}
    Upgrade -->|Yes| Options
    Upgrade -->|No| Cart
    Options --> Cart[장바구니 확인]
    Cart --> ServiceSplit{서비스 유형}
    ServiceSplit -->|딜리버리| OrderD[주문하기: 배달정보·주문정보 확인]
    ServiceSplit -->|징거벨오더| OrderZPre[주문하기: 매장정보 확인]
    OrderZPre --> Pickup{수령방법}
    Pickup -->|바로방문| Arrive[매장 도착 확인]
    Pickup -->|예약방문| Reserve[예약방문 시간 선택]
    Arrive --> OrderZ[쿠폰 선택 팝업]
    Reserve --> OrderZ
    OrderD --> Pay[결제수단 선택]
    OrderZ --> Pay
    Pay --> PayGo[결제 진행]
    PayGo --> PayResult{결제 성공?}
    PayResult -->|실패| Pay
    PayResult -->|성공| Done([주문완료])
    Done --> Status[주문현황<br/>주문→조리→배달/픽업→완료]
```
