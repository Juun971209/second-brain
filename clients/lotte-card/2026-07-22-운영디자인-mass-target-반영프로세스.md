## 핵심 요약

롯데카드 운영 디자인 업무에서 이벤트 화면(UI)이 최신 가이드로 업데이트되지 않는 이유와, 신규 등록 시 카테고리 분류가 이원화되는 이유를 정리. 퍼블팀/카드비즈팀이 화면을 직접 챙기지 않기 때문에 실무적으로 디자인팀이 손댄 이벤트만 업데이트되는 구조.

## 상세 내용

**1. 이벤트 카테고리 분류 이원화**
- 티켓/업무시트에는 유하나 책임 요청으로 카드 관련 이벤트를 모두 "카드프로모션"으로 통일해서 기재
- 실제 개발계/운영계 등록 시에는 담당 부서가 정한 실제 카테고리로 등록
- 즉 티켓 상 분류와 시스템 등록 분류가 다름 — 헷갈리지 않도록 주의 필요

**2. 이벤트 화면(UI)이 최신화되지 않는 이유**
- 원칙적으로는 디자인 가이드 개편 후 퍼블팀이 기존 화면을 업데이트해야 함
- 하지만 카드비즈팀과 퍼블팀이 화면을 직접 챙기는 스타일이 아니라서, 실제로는 **디자인팀이 손댄(수정한) 이벤트만** 최신 가이드로 업데이트됨
- MASS(전체 대상) 이벤트: 노출 대상이 많아 디자인을 수정해서 전달
- TARGET(특정 고객 대상) 이벤트: 디자인팀을 거치지 않고 진행되는 경우가 많고, 일부 고객만 보므로 우선순위 낮게 두고 그대로 방치
- 작업 유입 경로에 따라 처리 방식 분기:
  - 퍼블팀에서 작업하다 운영배포대기 단계로 넘어온 경우 → 그대로 운영계에만 반영
  - 시간 여유가 있어 디자인팀으로 넘어온 경우 → 참여방법·버튼 색상·버튼 이름(예: "카드 혜택 보기") 등을 수정해서 전달

**3. 외부 제휴사 게시 관련 용어**
- 게시번호를 새로 만들어 유지하는 경우도 있고, 이미지만 만드는 경우도 있음
- 티켓에 "외부제휴사 게시"라고 적힌 것은 시스템 상 **Target = Shadow**를 의미 (단, 업무시트에는 "Target"으로 표기)

**4. 외부제휴사 배너 작업 체크리스트**
- 신규카드 여부 확인 → 신규카드면 고해상도 카드 이미지 별도 요청
- 물량이 많거나 디테일 요구사항이 있으면 제휴사 가이드 요청 필요
- 내부 가이드가 없는 배너이므로, 대상 디바이스(모바일/PC/기타)를 가장 먼저 확인해야 함
